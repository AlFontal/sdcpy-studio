    SDC Map methodology

    What the app actually does

    1. Align the driver series and the gridded field on the same valid time axis for the selected date window and field subset.
    2. Detect local positive and negative extrema in the driver, rank them by magnitude, enforce a minimum separation of r_w, and keep up to N+ and N-.
    3. Build the base state from aligned time steps where |driver| < beta * x0, with x0 equal to the weakest selected event magnitude. Selected event windows are excluded from that baseline.
    4. Subtract the base-state mean field from the aligned field so the mapped response is measured relative to the inferred normal state.
    5. For each selected event and each grid cell, extract event-local fragments of width r_w and test lagged driver-vs-field correlations only inside that local event neighborhood.
    6. Use the permutation test configured in the UI to keep only significant event-local correlations at the selected alpha level.
    7. Average the retained event-local summaries separately for the positive and negative event classes.
    8. Render the active class as static A/B/C/D summary maps plus a lag-resolved explorer. Time units are inferred from the actual dataset cadence and shown as days, weeks, months, years, or generic time steps.

    How to read the outputs

    A is the strongest retained correlation, B is the retained driver-fragment position relative to the event peak, C is the retained lag between driver and field fragments, and D is the retained field timing relative to the event peak.

    This follows the event-conditioned workflow documented in docs/sdc_explained.md and matches the current package implementation.
